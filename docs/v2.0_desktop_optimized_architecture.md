# Đánh Giá Kiến Trúc & Đề Xuất Tối Ưu Cho Desktop App (v2.0)

## 1. Đánh giá hai phiên bản tiền nhiệm

### 1.1 Phiên bản 1.0 (Clean Architecture / Layered)
**Đặc điểm:** Tách biệt rõ ràng theo chiều ngang: `presentation` (UI), `core` (Logic), `modules` (Interfaces), `plugins` (Implementations).
*   **Ưu điểm:**
    *   Cấu trúc rất chuẩn mực, dễ hiểu đối với lập trình viên quen với MVC/MVVM truyền thống.
    *   Tầng UI (`presentation`) tập trung, dễ quản lý styling (CSS/QSS) đồng bộ.
*   **Nhược điểm đối với Desktop App:**
    *   **Phát triển phân mảnh:** Để thêm một tính năng (VD: Bóc băng video), lập trình viên phải nhảy qua lại giữa quá nhiều thư mục (Views -> ViewModels -> Core Logic -> Modules).
    *   **Khó mở rộng theo tính năng:** Khi ứng dụng lớn lên, tầng UI sẽ trở thành một "bình chứa" khổng lồ, phức tạp và khó gỡ rối.

### 1.2 Phiên bản 1.5 (Hybrid Variant - Vertical Slice)
**Đặc điểm:** Chia cắt tầng Application thành các khối tính năng dọc (Features) như `generate_subtitles`, `full_auto_dubbing`. Các tầng lõi bên dưới giữ nguyên.
*   **Ưu điểm:**
    *   Khả năng cô lập cực tốt. Phát triển/bảo trì một tính năng không sợ ảnh hưởng đến tính năng khác.
    *   Mã nguồn gắn liền chặt chẽ với "Use Case" (nghiệp vụ kinh doanh), dễ dàng định vị lỗi.
*   **Nhược điểm đối với Desktop App:**
    *   **Quản lý State khó khăn:** Desktop app thường cần chia sẻ trạng thái giữa các màn hình (VD: Cấu hình chung, tiến trình đang chạy ẩn). VSA thuần túy dễ dẫn đến việc các tính năng bị "mù" thông tin của nhau.
    *   **Quản lý Luồng (Threading):** Trong Python Desktop (PyQt/PySide), việc mỗi Feature tự quản lý UI và gọi lệnh xuống Core rất dễ vô tình làm treo (block) Main UI Thread nếu không thiết kế luồng cực kỳ cẩn thận.

---

## 2. Đề Xuất Phiên Bản 2.0: Event-Driven Desktop Hybrid Architecture

Dựa trên ưu khuyết điểm của v1.0 và v1.5, cùng với đặc thù của một Desktop App bằng Python (PySide6/PyQt6) thường phải chạy các tác vụ AI cực nặng, mình đề xuất **Phiên bản 2.0: Event-Driven Desktop Hybrid Architecture**.

### 2.1 Các nguyên tắc tối ưu cốt lõi (Core Optimizations)

1.  **Centralized State & Reactive UI (Quản lý Trạng thái Tập trung):** Thay vì mỗi Feature tự ôm một trạng thái, chúng ta sử dụng một Global Store (tương tự Redux/Vuex). Giao diện chỉ việc "lắng nghe" (subscribe) vào Store này để tự động cập nhật.
2.  **Strict Thread Boundary (Ranh giới Luồng nghiêm ngặt):**
    *   **UI Thread (Main):** CHỈ dùng để render giao diện.
    *   **Worker Threads / Sub-processes:** Tất cả tác vụ liên quan tới Core (Load model, Infernce, Xử lý file) bắt buộc chạy ở tiến trình con hoặc luồng khác. Giao tiếp với UI thông qua hệ thống Event Bus/Signals.
3.  **Feature as a Plugin (Tính năng như một Plugin của UI):** Các Vertical Slices ở v1.5 giờ đây sẽ đóng vai trò như các module cắm-rút (plug-and-play) vào một Window Shell chính.

    │   └── common_widgets/                 # Các component dùng chung (Loading, Dialog, Waveform)

    │   ├── voice_studio/
    │   │   ├── view.py
    │   │   └── actions.py
    │   │

    │   ├── event_bus/                      # [MỚI] Xương sống giao tiếp giữa Features và Core

### 2.3 Vì sao v2.0 lại tối ưu nhất cho Python Desktop?

1.  **Không bao giờ đơ màn hình (Non-blocking UI):** Bằng cách ép buộc các `features` giao tiếp với `core` thông qua `event_bus` (gửi lệnh và lắng nghe tín hiệu phản hồi thay vì gọi hàm trực tiếp chờ kết quả), UI sẽ luôn mượt mà kể cả khi model Whisper đang load VRAM mất 30 giây.
2.  **Dễ dàng chia nhỏ màn hình phức tạp:** `ui_shell/global_store.py` giữ toàn bộ trạng thái. Ví dụ tính năng "Lồng tiếng" yêu cầu chọn giọng từ "Phòng thu giọng", nhờ Global Store, tính năng lồng tiếng dễ dàng lấy được danh sách giọng mà không cần viết các hàm móc nối phức tạp giữa các lát cắt VSA.
3.  **Tái sử dụng cao hơn V1.5:** Đưa các UI dùng chung ra `ui_shell/common_widgets`, loại bỏ sự trùng lặp có thể xảy ra trong các `features` của v1.5.
4.  **Mở rộng tính năng cực nhanh:** Muốn thêm tính năng mới, chỉ cần tạo folder mới trong `features`, định nghĩa `view` (giao diện) và `actions` (lệnh gửi xuống core), sau đó đăng ký vào thanh menu của `ui_shell/main_window.py`.
